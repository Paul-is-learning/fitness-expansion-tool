📚 Methodologie SAZ v3 — 3 piliers
Le SAZ repose sur les 3 criteres cles de succes d'un club fitness a fort traffic :
SAZ = FLUX33% + DENSITE33% + JEUNESSE33%
FLUX = Metro30% + Malls25% + Bureaux25% + Universites10% + Pietons10%
DENSITE = Pop. cible30% + Densite/km225% + Pouvoir achat25% + Concurrence inv.20%
JEUNESSE = Universites40% + Age quartier30% + Nvx residentiels20% + Prix immo10%
Sources: INS Census 2021, Metrorex (750k pax/jour), imobiliare.ro, Overpass API, worldclass.ro, stayfit.ro. Donnees estimees = est.

Gap Analysis

🛠 Score de menace (0-100)
Chaque concurrent est score selon :
Menace = Proximite40% + Taille25% + Positionnement20% + Notoriete15%
<500m = menace max (100) | >3km = menace min (10). Les enseignes premium sur le meme segment cible que FP (low-cost premium) sont les plus menacantes.

SURFACES (25 verifiees)

ESTIMATION MEMBRES (double methode)
• Methode A — Surface × ratio: Premium 0.85 mbr/m2, Mid 1.1, Low-cost 1.5. Calibre sur WC (84k/45 clubs/~2000m2 moy = 0.93)
• Methode B — Google Reviews / anciennete × 81 × facteur_age. Calibre sur WC (219 avis moy / 9.5 ans = 23/an → 81× = 1,867 mbr). Sources: top-rated.online, citymaps.ro, wheree.com
• Resultat = moyenne(A, B) quand les 2 disponibles
HEATMAP DENSITE
• 397,009 batiments extraits de l'export OSM complet Roumanie (Geofabrik, 301MB)
• 194,090 residentiels traites avec ponderation volumetrique (emprise × etages)
• building:levels tag OSM (47,647 batiments = 12%) + estimation par type
• Calage census par secteur (facteur 0.005-0.009)
• Landuse OSM: filtrage residential vs commercial/industrial
• 3 modes: Nuit (residentiel), Jour (bureaux+unis+malls), Combine
• Pipeline: Geofabrik → osmium extract → osmium tags-filter → osmium export → Python aggregation

• Ramp-up: courbe S M1-M12 (8%→93%). Sources: benchmark FP France

⚠ Limites & avertissements
• Les estimations de membres sont des approximations basees sur des proxies (surface, reviews). Aucune donnee reelle publiee par les enseignes roumaines
• Les Google Reviews sont des estimations collectees via sites aggregateurs, pas scraping direct Google
• Les taux de captage (15-25%) sont des hypotheses sectorielles, pas des donnees mesurees en Roumanie
• La heatmap densite represente le volume bati (emprise × etages), pas le nombre reel de residents
• Les ages des clubs sont estimes par marque quand la date d'ouverture exacte n'est pas connue
• Population reelle x1.7 est une estimation — la realite pourrait etre x1.5 a x2.0
• Le modele walk-in (0.7% conversion) est un benchmark europeen, pas specifique Roumanie
